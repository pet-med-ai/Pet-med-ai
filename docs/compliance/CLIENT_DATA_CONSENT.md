# 客户数据使用同意书 V1

> 阶段定位：本文件是 Pet-Med-AI 客户数据使用同意书草案，用于前台打印、电子入院表单、旧病例导入和 AI 训练数据治理的统一口径。  
> 正式启用前必须交由法律 / 合规负责人复核，并按当地法规、诊所制度和实际数据处理方式修订。

## 1. 使用场景

适用于瀚森宠物医院在以下场景中征得宠物主人知情同意：

```txt
门诊 / 住院病例
影像 / 检验数据
电子病历文本
临床照片 / 术中视频
诊断标签与临床操作记录
AI 辅助问诊与质量改进数据
旧系统历史病例迁移后的数据治理
```

## 2. 可使用的数据类型

客户可选择是否允许医院在脱敏 / 去标识化后，将以下数据用于院内质量改进、临床辅助工具开发和 AI 研究：

```txt
[ ] 影像：X 光 / DR / CT / MRI / 超声 / 内窥镜等
[ ] 实验室检验结果：血常规、生化、电解质、尿检、血气等
[ ] 电子病历文本：主诉、病史、体检、诊断、用药记录、出院小结等
[ ] 临床照片 / 术中视频
[ ] 诊断标签、操作记录、治疗过程摘要
```

默认不应包含：

```txt
主人姓名
主人电话
证件号
详细住址
可直接识别个人身份的信息
```

## 3. 脱敏原则

数据进入训练池、测试集、研究集或外部协作前，应执行以下脱敏处理：

```txt
1. 去除主人姓名、电话、证件号、详细地址等直接身份信息。
2. 病例编号映射为不可逆 token 或随机标识。
3. DICOM / 影像文件头中的 PatientName、PatientID 等字段清除或替换。
4. 文件名中如含主人或宠物敏感标识，应替换为随机 token。
5. 自由文本病历中如出现主人电话、地址等信息，应在导出前自动或人工清理。
6. 脱敏脚本与导出批次应保留审计记录。
```

## 4. 同意与不同意

宠物主人可以选择：

```txt
[ ] 同意去标识化使用
[ ] 不同意去标识化使用
```

不同意不影响诊疗、收费、服务质量和医生诊疗决策。

## 5. 撤回权利

宠物主人可随时撤回同意。撤回方式：

```txt
前台登记
电话登记
微信 / 在线表单
书面申请
```

撤回后处理原则：

```txt
1. 收到撤回请求后，应记录 opt-out registry。
2. 30 日内停止将该客户 / 该病例的新数据用于训练或导出。
3. 90 日内将此前可移除的数据从训练池、导出集、待标注集中标记为不可用。
4. 已经参与过模型训练的模型权重，通常无法按个体完全回溯删除；但该个体数据应从后续训练、评估、导出和共享中排除。
```

## 6. 数据保留建议

正式启用前由医院管理层和合规负责人确认：

```txt
原始入站资料：建议内部留存 3 年或依病历管理制度执行
脱敏训练集：建议受控存储 3 年
不可变审计日志：建议保存不少于 5 年或按法规要求更长
撤回登记记录：建议不少于 5 年
```
