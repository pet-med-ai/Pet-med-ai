# Clinical Docs Export / Word Template Spec V1

## Stage

This stage defines the first Pet-Med-AI clinical document export specification for Word-compatible templates.

It is a specification and mapping stage only.

It does not:

```txt
generate real DOCX files
generate PDF files
add backend export APIs
change database schema
write Case records
enable EMR real import
modify existing clinical data
```

## Purpose

Pet-Med-AI needs printable clinical documents that can be generated from Case data, EMR webhook payloads, and future hospitalization workflows.

V1 defines two initial Word template families:

```txt
Admission / Hospitalization Record
Discharge Summary
```

Both should be bilingual:

```txt
Chinese label + English label
```

Both should support:

```txt
A4 page
2.54 cm margins
dark green header
right-top electronic stamp placeholder
double-language labels
{{placeholder_key}} template syntax
audit footer
```

## Visual standard

### Page

```txt
Paper: A4 portrait
Margins: 2.54 cm
Header height: about 35 mm
Body grid: 12-column invisible layout
Body font size: 10.5 - 11 pt
Line height: 1.35
```

### Brand colors

```txt
Primary dark green: #0F3B2E
White: #FFFFFF
Text dark gray: #2B2B2B
Background gray: #F5F7F6
```

### Stamp placeholder

```txt
Position: top right
Size: 120 x 120 px
Purpose: electronic clinic stamp image slot
Behavior: placeholder only in V1
```

Do not embed a real official stamp in template source control.

## Template syntax

Use Jinja-style / docxtpl-compatible placeholders:

```txt
{{case_id}}
{{pet.name}}
{{owner.name}}
{{clinician.name}}
{{timestamp}}
```

Nested dotted keys are allowed in the spec and must be normalized by the future renderer.

## Audit footer

Every generated document must include:

```txt
Generated by {{generator}}
Clinician ID {{clinician_id}}
Generated at {{timestamp}}
Hash {{hash}}
```

The hash should be computed from a canonicalized document context in a future implementation.

Example future hash logic:

```txt
hash = sha256(canonical_json(context)).hexdigest()[:12]
```
