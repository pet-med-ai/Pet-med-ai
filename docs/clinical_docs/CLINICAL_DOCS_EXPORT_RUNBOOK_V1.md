# Clinical Docs Export Runbook V1

## Purpose

This runbook describes how future Pet-Med-AI clinical document export should work using Word-compatible templates.

This V1 is a spec only. No DOCX is generated by this stage.

## Future export flow

```txt
User opens Case detail
User selects document type
Backend builds document context
Backend validates required keys
Backend renders DOCX from template
Backend computes context hash
Backend writes optional audit record in later stage
User downloads DOCX or PDF
```

## Future backend technology options

Preferred Python implementation:

```txt
docxtpl
python-docx
LibreOffice headless for optional PDF conversion
```

Possible Node implementation:

```txt
docxtemplater
pizzip
LibreOffice / office-convert service for PDF
```

## Required security rules

```txt
Do not embed real clinic seal in source control
Do not expose SECRET_KEY or DATABASE_URL in documents
Do not include raw webhook payload unless explicitly selected
Do not write structured prescriptions from document export
Do not download attachments in document export V1
Do not create or update Case during export
```

## Required audit fields

Every generated document context should include:

```txt
generator
clinician_id
timestamp
hash
case_id
template_id
template_version
```

## Future API design draft

```txt
POST /api/clinical-docs/render
```

```json
{

  "template_id": "admission_hospitalization_record_bilingual",
```
