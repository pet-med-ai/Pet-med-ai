# 运维 KPI 仪表口径与原型 V1

> 阶段定位：本文件只定义临床 + 运营 KPI 的口径、阈值、页面布局和后续落地路线。  
> 当前 V1 不连接真实数据库，不新增表，不新增后端 API，不改前端运行页面。

## 1. 目标

把“数据迁移后的质量”和“日常医疗执行质量”统一到一页仪表盘原型中，用于：

- 班前 / 班后两分钟自检。
- 周例会复盘。
- 旧病例导入后的质量监测。
- 后续医院设备、影像、检验、随访、QA 数据接入后的运营看板。

## 2. 当前边界

本阶段只做：

```txt
KPI 口径
阈值
可视化布局
mock 数据
SQL 草案
后续 API 草案
```

本阶段不做：

```txt
真实 SQL 查询
真实后端 API
真实前端路由
数据库新增字段 / 新表
Alembic revision
QA 自动工单
收费 / 影像 / 随访真实接入
```

## 3. 六个核心 KPI

### 3.1 病例字段完整度率 Case Completeness

**口径**  
本期内“已就诊并建档病例”中，关键字段全部非空的占比。

V1 建议关键字段：

```txt
主诉
体温
体重
过敏史
主要检查
诊断
医嘱 / 处方
收费单
```

Pet-Med-AI 当前可先用现有字段近似：

```txt
chief_complaint
weight
exam_findings
analysis
prognosis 或 treatment
```

等后续诊断数据模型 / 处方 / 收费模型建立后，再升级为完整口径。

**建议阈值**

```txt
>= 95%
```

**可视化**

```txt
顶部大数字卡
医生 / 科室热力图
异常病例明细表
```

---

### 3.2 影像复拍率 Repeat Imaging Rate

**口径**  
同一病例在 24–72 小时内，对同一部位、同一模态进行二次以上成像的比例。排除“计划性复查”。

**建议阈值**

```txt
< 8%
```

**可视化**

```txt
顶部 KPI 卡
时序折线
复拍异常明细表
```

**注意**  
当前 Pet-Med-AI 尚未建立 `imaging` 表，所以 V1 只定口径，不接真实数据。

---

### 3.3 回访合规率 Follow-up Compliance

**口径**  
到期日当天 ±1 天内完成指定回访动作的占比。

建议分档：

```txt
当天完成
逾期 1 天完成
逾期 2 天完成
未完成
```

**建议阈值**

```txt
>= 85%
```

**可视化**

```txt
分段条形图
责任人看板
逾期回访 Top20
```

---

### 3.4 平均结案时长 Average Time-to-Close

**口径**  
病例从建档到结案 / 出院 / 结算完成的平均小时数。

**建议阈值**

```txt
门诊 <= 6 小时
住院 <= 72 小时
```

**可视化**

```txt
顶部 KPI 卡
时序折线
分布箱线图
超长结案 Top20
```

---

### 3.5 重复影像占比 Duplicate Imaging Share

**口径**  
当期影像收入中，标记为“重复 / 无新增诊断价值”的项目金额占比。

**建议阈值**

```txt
< 5%
```

**可视化**

```txt
环形占比卡
医生 / 部位分解柱状图
重复收费明细表
```

**注意**  
这个 KPI 依赖收费与影像标注模型，当前 V1 只定义口径。

---

### 3.6 审计覆盖率 QA Audit Coverage

**口径**  
当期病例中被纳入 QA 抽检清单的占比。

**建议阈值**

```txt
常规 >= 15%
新医生 / 重点科室 >= 30%
```

**可视化**

```txt
顶部 KPI 卡
抽检清单表
严重问题分布
医生 / 科室覆盖率排行
```

## 5. 班次节奏

### 班前 / 班后 2 分钟自检

只看两张卡：

```txt
病例字段完整度率
影像复拍率
```

低于阈值时，立即打开异常触发表。

### 周例会 15 分钟

重点看：

```txt
平均结案时长趋势
回访合规率趋势
QA 抽检问题分布
影像复拍 / 重复影像明细
```

### 质量闭环

后续可把异常行转为 QA 任务：

```txt
负责人
截止时间
关联病例 / 影像 / 回访
处理状态
复盘备注
```

## 6. 最小数据模型草案

后续真实接入时，建议至少规划这些表：

```txt
cases
imaging
imaging_billing
followups
qa_audit
```

当前 Pet-Med-AI 已有 `cases`，其余表需要在后续 KPI 数据模型 V1 中通过 Alembic 新增。
