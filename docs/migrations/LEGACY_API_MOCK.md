# Legacy Case Import API Mock V1

This stage adds a non-mutating API mock for legacy case imports.

## Endpoint

```txt
POST /api/migrations/legacy-cases/mock
```

Authentication is required.

## Scope

The mock endpoint accepts records generated by:

```bash
python3 scripts/legacy_cases_to_case_payloads.py docs/migrations/LEGACY_CASES_IMPORT_TEMPLATE.csv \
  --jsonl-out /tmp/legacy_case_payloads.jsonl
```

Then send the JSONL records as:

```json
{
  "batch_id": "pilot-001",
  "records": [
    {
      "operation": "case_create",
      "dry_run": true,
      "legacy_case_id": "HS-2026-000123",
      "idempotency_key": "...",
      "case_create": {}
    }
  ]
}
```

## Guarantees

V1 is intentionally safe:

```txt
writes_database = false
calls_case_create_api = false
no Case rows are created
no legacy data is persisted
```

The response only validates and previews the future import contract.

## Response

```json
{
  "message": "mocked",
  "mode": "api_mock",
  "dry_run": true,
  "writes_database": false,
  "calls_case_create_api": false,
  "received": 1,
  "accepted": 1,
  "rejected": 0,
  "items": []
}
```

## Validation

The mock rejects rows with:

```txt
missing operation=case_create
dry_run not true
missing idempotency_key
duplicate idempotency_key in the same batch
missing case_create
missing case_create.patient_name
missing case_create.species
missing case_create.chief_complaint
```

## Next stage

After this mock passes local and online smoke, the next safe stage is:

```txt
Legacy Case Import API dry-run V2
```

That stage may add a server-side dry-run endpoint that accepts JSONL-like records and returns a batch report, still without writing the production database.
